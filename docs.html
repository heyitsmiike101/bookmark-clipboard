Introduction

Everything in the dashboard — bookmarks, categories, clipboard entries and file attachments — is controllable through a small HTTP API. This page documents every action.

All endpoints live under a single path, /api, with the operation chosen by an ?action= query parameter (for example /api?action=config). The server is plain Node.js.

Base URLhttp://localhost:3000 (this server: )
FormatRequest bodies and responses are JSON, except upload (multipart) and download (binary).
CORSOpen (Access-Control-Allow-Origin: *) so other tools and agents can call it.
ErrorsNon-2xx responses return { "error": "message" }.
AuthNone. Run it on a trusted LAN, or put a reverse proxy / auth layer in front.
Every change is written to config.json and bumps a version counter. Poll GET version to detect changes made by other clients.

Data model

The whole dashboard state is one JSON document:

{
  "bookmarks": {
    "Category Name": [
      { "id": "bm_abc123", "name": "Plex", "url": "https://app.plex.tv", "ping": true },
      { "id": "bm_def456", "name": "Server IP", "info": "192.168.1.10" }
    ],
    "Another Category": [ ... ]
  },
  "clips": [ ... ],
  "version": 42
}
ConceptNotes
CategoryA key in the bookmarks object. Category order is the key order and is preserved.
BookmarkAn item in a category array. Order within a category is the array order.
idStable, server-assigned (bm_…). Use it as the reliable handle for edits. Auto-assigned to any bookmark missing one.
nameDisplay label. required
Link tileHas url, optionally ping: true (shows an online/offline dot).
Info tileHas info instead of url — a click-to-copy value (IP, key, note). Cannot have ping.
A bookmark is either a link (url) or an info tile (info) — never both. Setting one via update-bookmark clears the other.

Locating a bookmark

update-bookmark, delete-bookmark and move-bookmark need to identify a target bookmark. Provide a locator — any one of these (checked in this order):

LocatorExampleNotes
id{ "id": "bm_abc123" }Preferred. Unambiguous, searches all categories.
category + index{ "category": "ARRs", "index": 0 }0-based position within the category.
category + name{ "category": "ARRs", "name": "Sonarr" }First bookmark with that name in the category.

A locator that matches nothing returns 404.

Using with an AI agent

The API is designed to be driven by an LLM/agent. Recommended loop:

  1. Read first. GET /api?action=bookmarks to see current categories, bookmarks and their ids.
  2. Mutate by id. Prefer ids returned from the read for edits/moves — robust against duplicate names.
  3. Re-read after multi-step organizing if you need fresh indices/ids.

For bulk reorganization (re-categorizing many links at once), the simplest path is: read the bookmarks object, rearrange it in memory, and write it back with one POST bookmarks call. For incremental edits, use the granular actions below.

Agent tip: the granular endpoints are intentionally small and predictable so each can map to one tool call. A minimal toolset is: add-bookmark, update-bookmark, delete-bookmark, move-bookmark, plus add-category / rename-category / delete-category.

Read endpoints

GET /api?action=config

Return the entire document: bookmarks, clips and version.

curl "$BASE/api?action=config"
GET /api?action=bookmarks

Return just the bookmarks tree plus the current version — the usual starting point for an agent.

curl "$BASE/api?action=bookmarks"

→ { "bookmarks": { "ARRs": [ { "id": "bm_…", "name": "Sonarr", "url": "…" } ] }, "version": 42 }
GET /api?action=version

Return { "version": N }. Increments on every change — poll it to detect updates.

GET /api?action=health

Runtime + storage diagnostics (Node version, data dir, read/write status, upload limit).

Bookmark actions

POST /api?action=add-bookmark

Create a bookmark in a category (the category is created if it doesn't exist).

FieldTypeNotes
categorystringrequiredTarget category; created if missing.
namestringrequiredDisplay label.
urlstringone ofMakes a link tile.
infostringone ofMakes an info (copy) tile. Provide url or info.
pingbooleanoptionalOnly for link tiles; enables the status dot.
indexnumberoptionalInsert position (default: end of category).
curl -X POST "$BASE/api?action=add-bookmark" \
  -H "Content-Type: application/json" \
  -d '{ "category": "AI Tools", "name": "Claude", "url": "https://claude.ai", "ping": true }'

→ { "ok": true, "bookmark": { "id": "bm_…", "name": "Claude", "url": "https://claude.ai", "ping": true },
    "category": "AI Tools", "index": 0 }
POST /api?action=update-bookmark

Edit a bookmark's fields. Identify it with a locator; put new values in fields.

FieldNotes
locatorid, or category+index, or category+name.
fields.nameRename the bookmark.
fields.urlSet/replace the URL (converts an info tile to a link tile).
fields.infoSet/replace the info value (converts a link tile to an info tile).
fields.pingtrue/false for link tiles.
curl -X POST "$BASE/api?action=update-bookmark" \
  -H "Content-Type: application/json" \
  -d '{ "id": "bm_abc123", "fields": { "name": "Claude (Opus)", "ping": false } }'
POST DELETE /api?action=delete-bookmark

Remove a bookmark. Use a locator in the JSON body (POST) or as query params (DELETE).

# By id (POST body)
curl -X POST "$BASE/api?action=delete-bookmark" \
  -H "Content-Type: application/json" -d '{ "id": "bm_abc123" }'

# By category + name (DELETE with query params)
curl -X DELETE "$BASE/api?action=delete-bookmark&category=ARRs&name=Sonarr"

→ { "ok": true, "removed": { "id": "bm_…", "name": "Sonarr", "url": "…" }, "category": "ARRs" }
POST /api?action=move-bookmark

Move a bookmark to another category and/or a new position. Same-category moves reposition it.

FieldNotes
locatorrequiredWhich bookmark to move.
toCategoryoptionalDestination category (created if missing). Default: same category.
toIndexoptionalPosition in the destination (default: end).
curl -X POST "$BASE/api?action=move-bookmark" \
  -H "Content-Type: application/json" \
  -d '{ "id": "bm_abc123", "toCategory": "Favorites", "toIndex": 0 }'
POST /api?action=reorder-bookmarks

Set the order of bookmarks within one category. order is a list of ids, names, or indices. Any bookmarks you omit are appended after, keeping their original relative order.

curl -X POST "$BASE/api?action=reorder-bookmarks" \
  -H "Content-Type: application/json" \
  -d '{ "category": "ARRs", "order": ["bm_radarr", "bm_sonarr", "Prowlarr"] }'
POST /api?action=bookmarks

Replace the entire bookmarks tree at once — ideal for bulk reorganizing. Existing ids you include are preserved; any bookmark without one is assigned a fresh id.

curl -X POST "$BASE/api?action=bookmarks" \
  -H "Content-Type: application/json" \
  -d '{ "Work": [ { "name": "Jira", "url": "https://jira.example.com" } ],
        "Home": [ { "name": "Router", "info": "192.168.1.1" } ] }'

→ { "ok": true, "bookmarks": { … with ids … }, "version": 43 }

Category actions

POST /api?action=add-category

Create a new, empty category. Errors with 409 if it already exists.

categoryrequiredName of the new category.
indexoptionalPosition among categories (default: end).
curl -X POST "$BASE/api?action=add-category" \
  -H "Content-Type: application/json" -d '{ "category": "AI Tools", "index": 0 }'
POST /api?action=rename-category

Rename a category, preserving its position and bookmarks.

curl -X POST "$BASE/api?action=rename-category" \
  -H "Content-Type: application/json" -d '{ "category": "AI Tools", "newName": "AI" }'
POST DELETE /api?action=delete-category

Delete a category and all bookmarks in it.

curl -X POST "$BASE/api?action=delete-category" \
  -H "Content-Type: application/json" -d '{ "category": "AI" }'

# or:  curl -X DELETE "$BASE/api?action=delete-category&category=AI"
→ { "ok": true, "category": "AI", "removedCount": 3 }
POST /api?action=reorder-categories

Set the order of categories. Omitted categories are appended in their original order.

curl -X POST "$BASE/api?action=reorder-categories" \
  -H "Content-Type: application/json" \
  -d '{ "order": ["Favorites", "Work", "Home"] }'

Clipboard & file actions

The clipboard holds shared text snippets and uploaded files. A clip with todayOnly: true is auto-removed after the day it was created.

POST /api?action=clip

Add a text clip.

curl -X POST "$BASE/api?action=clip" \
  -H "Content-Type: application/json" \
  -d '{ "text": "shared note", "todayOnly": false }'

→ { "id": "…", "type": "text", "text": "shared note", "todayOnly": false, "createdAt": "…" }
POST /api?action=upload multipart/form-data

Upload a file as a clip. Fields: file (the file) and optional todayOnly (1/0). No size limit — any size, including empty files.

curl -X POST "$BASE/api?action=upload" \
  -F "file=@/path/to/report.pdf" -F "todayOnly=0"

→ { "id": "…", "type": "file", "filename": "…_report.pdf", "originalName": "report.pdf",
    "size": 12345, "todayOnly": false, "createdAt": "…" }
POST /api?action=update-clip&id=ID

Edit the text of an existing text clip.

curl -X POST "$BASE/api?action=update-clip&id=123" \
  -H "Content-Type: application/json" -d '{ "text": "updated note" }'
DELETE /api?action=clip&id=ID

Delete a clip. For file clips, also pass filename to remove the stored file from disk.

curl -X DELETE "$BASE/api?action=clip&id=123&filename=123_report.pdf"
GET /api?action=download&filename=F&original=NAME

Download a stored file. filename is the stored name; original sets the suggested download filename.

curl -OJ "$BASE/api?action=download&filename=123_report.pdf&original=report.pdf"

Tip: set BASE in your shell first — BASE="http://localhost:3000" — so the examples above run as-is.
